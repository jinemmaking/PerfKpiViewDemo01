2. 技术规格说明

2.1 前端核心选型 (Recommended for Cursor)
- 基础库: Vue 3 (Script Setup) + Vite
- 组件库: Element Plus (提供 Tree, Table, Radio, Tag 等企业级组件)
- 图表: ECharts 5.x (配置 tooltip 联动和 smooth 曲线)
- 布局: Tailwind CSS (快速实现左右分栏及内边距控制)
  
2.2 数据详情表设计 (Data Table)
表格位于图表下方，用于展示数值统计结果：
- 列定义:
  1. 名称: 端口名称；若处于“求和/平均”模式，则显示“汇总结果”。
  2. 最大值 (Max): 当前时间段内的峰值。
  3. 最小值 (Min): 当前时间段内的谷值。
  4. 平均值 (Avg): 当前时间段内的算术平均值。
- 样式: 
  - 使用 el-table 的 stripe (斑马纹) 提高阅读性。
  - 数值列建议右对齐，方便对比量级。

3. UI 交互逻辑

3.1 侧边栏 (Sidebar)
- 搜索: 实时过滤树节点。
- 限制: 勾选逻辑需在 onCheckChange 中拦截，超过 10 个端口时，通过 ElMessage.warning 提醒用户。
- 已选标签: 点击标签上的 x 自动取消树中对应的勾选状态。
  
3.2 模式切换响应
- 原始值模式: 
  - 图表渲染 N 条曲线。
  - 表格渲染 N 行统计数据。
- 求和/平均模式: 
  - 图表渲染 1 条曲线。
  - 表格仅渲染 1 行数据，标题为“已选端口汇总(Sum)”或“已选端口平均(Avg)”。
    
3.3 ECharts 配置优化
- AreaStyle: 给曲线下方增加淡色渐变，增加视觉“体量感”。
- MarkLine: 在图表中通过横向虚线实时标注当前选中曲线的“平均值”水位。

4. 给 Cursor 的开发任务指令 (您可以分次输入)

任务 1：侧边栏与数据结构
“请使用 Vue3 和 Element Plus 实现页面左侧。包含一个带搜索的 el-tree 和下方的‘已选端口’标签区。模拟 10 个以上的设备数据，每个设备包含若干端口。实现勾选端口时，下方标签实时增加，且限制最多勾选 10 个。”

任务 2：控制面板与统计表
“在页面右侧顶部实现指标选择和计算模式切换按钮。在右侧底部实现一个 el-table。表格包含：端口名称、最大值、最小值、平均值四列。要求表格数据能根据左侧勾选的端口实时生成行。”

任务 3：ECharts 联动与聚合算法
“实现中间的 ECharts 区域。
1. 编写模拟数据函数，生成 1 分钟间隔的时序数据。
2. 实现核心逻辑：当模式切换为‘求和’时，将已选端口的所有数据点相加生成新数组并渲染成单条曲线；当切换为‘平均’时同理。
3. 统计表格中的 Max/Min/Avg 值必须由这些时序数据实时计算得出。
4. 确保切换模式时图表有过渡动画。”
