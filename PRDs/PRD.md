 (PRD)：企业级性能数据展示台

1. 项目背景
项目名称：企业级性能数据展示台 (Performance Data Aggregator)
目标用户：网络运维工程师
核心痛点：现有的 iSmartOne 系统无法对跨设备/跨端口的流量进行实时叠加统计。本工具旨在通过便捷的 UI 交互，实现多条线路性能指标的“复合展示”与“数学聚合计算”。

2. 总体功能要求
1. 多源选择：支持跨设备勾选端口，限制单次最多选择 10个 端口。
2. 动态聚合：支持“原始数据”、“多线求和”、“多线平均”三种计算模式。
3. 高性能渲染：采用时序图表展示流量趋势，具备最值标注。
4. 配置复用：支持保存选中的端口组合，方便下次一键调取。
  

---

3. 详细功能模块

3.1 资源检索与选择 (左侧面板)
- 交互形式：树形结构 (Tree)。
- 层级逻辑：组织机构 > 数据中心 > 机房 > 设备 > 端口。
- 搜索功能：支持模糊搜索设备名称或端口名称。
- 勾选约束：
  - 端口带复选框，最多允许勾选 10个。
  - 达到 10 个后，其余复选框变为禁用状态，并弹出气泡提示“最多支持10个端口叠加”。
- 已选预览：在树下方或顶部显示已选中的端口标签（Tag），支持点击“x”快速删除。
  
3.2 性能看板控制 (顶部工具栏)
- 指标选择：
  - 入流量 (In-Traffic)
  - 出流量 (Out-Traffic)
  - 入带宽利用率 (In-Usage)
  - 出带宽利用率 (Out-Usage)
- 计算模式 (核心逻辑)：
  1. 原始值 (Original)：图表展示 N 条曲线（N=选中数量）。
  2. 求和值 (Sum)：后端聚合，图表仅展示 1条 汇总曲线。
  3. 平均值 (Avg)：后端聚合，图表仅展示 1条 平均值曲线。
- 时间范围：支持快速选择（近1h、近6h、近24h）及自定义起止时间。
- 保存视图：点击可输入名称，保存当前 [端口列表 + 指标 + 计算模式] 的组合。
  
3.3 数据可视化 (图表区)
- 图表类型：折线面积图 (Line/Area Chart)。
- 数据标注：
  - Max/Min 值：根据当前展示的曲线，在图例旁边或图表显眼位置动态显示最大值和最小值。
  - 单位自适应：流量单位根据数值大小自动转换 (bps -> Kbps -> Mbps -> Gbps)。
- 交互体验：
  - 支持缩放 (DataZoom)。
  - 十字准星悬浮提示 (Tooltip)，同步展示该时间点所有参与计算的端口详情。

4. 技术实现规格

4.1 技术栈建议
- 前端框架：Vue 3 (SFC) + Vite
- UI 组件库：Element Plus (符合 iOne 风格)
- 图表库：ECharts 5.0+
- 样式：Tailwind CSS
  
4.2 数据模拟与聚合逻辑 (Mocking Strategy)
由于当前不编写后端代码，需在前端模拟以下逻辑：
1. Mock 接口：模拟一个 fetchPerformanceData 函数。
2. 模拟聚合：
  - 当用户选择“求和”或“平均”时，该函数应在内部对选中的多个端口数组进行 reduce 操作。
  - 算法说明：Result[time_t] = Port1[time_t] + Port2[time_t] + ...
3. 并发模拟：模拟 500ms-1000ms 的网络延迟，以展示 Loading 效果。
  
4.3 数据结构参考

5. UI/UX 设计规范
- 布局：左侧侧边栏 (20%) + 右侧主区域 (80%)。
- 配色：主色调使用 iSmartOne 标准蓝/橙色；多曲线颜色建议使用 ECharts 内置的高辨识度配色方案。
- 响应式：图表需随窗口大小自动 Resize。

6. 给 Cursor 的分步执行建议 (Prompt 建议)

1. Step 1: "请根据 PRD 搭建基础布局，使用 Vue3 和 Element Plus 实现左侧设备树和右侧 ECharts 空白画布，要求树支持搜索和最多10个端口的选择限制。"
2. Step 2: "实现模拟数据生成器，能够产生 10 个端口的时序数据。编写核心聚合算法函数，支持将多个端口的数据按时间戳进行 Sum 和 Avg 计算。"
3. Step 3: "将图表与控制面板关联。当用户切换‘指标’或‘计算模式’时，调用聚合函数并重新渲染 ECharts，确保图表包含最大/最小值的标注。"
4. Step 4: "添加‘保存视图’功能，使用 localStorage 存储用户保存的组合，并在页面顶部提供一个下拉菜单快速加载已保存的视图。"
